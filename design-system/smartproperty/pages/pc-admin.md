# Pc Admin Page Overrides

> **PROJECT:** SmartProperty（智慧物业）
> **Page:** PC 物业管理后台（管理员 / 物业管理人员）
> **Overrides:** `design-system/smartproperty/MASTER.md`

> 本文件规则 **优先于** Master。未列事项一律遵循 Master。

---

## 视觉与气质（覆盖 Master）

### Layout Overrides

- **整体结构：** 左侧主导航 + 顶栏（项目/小区切换、搜索、消息、账号）+ 主工作区；主区支持 **12 列栅格**，表格/看板可全宽。
- **内容最大宽度：** 数据密集型页面可 **全宽（≥1280px 舒适区）**；表单类详情可用 `max-w-5xl` 居中。
- **AI 助手：** **右侧抽屉/侧边栏**（可折叠），与主内容并存，不遮挡全局导航。

### Color Overrides

- **气质：** **科技智能感**（相对 Master 的偏社区暖色，后台可更冷、更「运维台」）。
- **建议实现：** 在 Master 紫绿体系上，后台主界面以 **slate / zinc 中性色** 为底（如背景 `#F8FAFC`～`#0F172A` 二选一主题），**品牌主色**仍用 Master 的 Primary 做强调；**状态色**固定语义：成功 `#22C55E`、警告 `#F59E0B`、危险 `#EF4444`、信息 `#3B82F6`。
- **深色主题（可选）：** 背景 `#0F172A`，表面 `#1E293B`，正文 `#F8FAFC`，主强调可用青绿或 Master CTA 绿。

### Typography Overrides

- **数据展示：** 指标数字、表格金额可用 **tabular figures**（`font-variant-numeric: tabular-nums`）。
- **中文：** 正文优先 **系统字体栈**（如 `PingFang SC`, `Microsoft YaHei`, `sans-serif`），英文可与 **Plus Jakarta Sans** 混排，避免全站仅依赖西文字体。

### Spacing Overrides

- **密度：** 列表/表格 **偏高密度**（行高紧凑、可切换「舒适/紧凑」）；仪表盘卡片区间距可用 `--space-lg`～`--space-xl`。

### Component Overrides

- **首页仪表盘：** **卡片 + 图表 + 数字看板** 混排；核心 KPI：入住率、待处理工单、营收、设备状态。
- **工单：** 列表 + **看板/时间线**（状态流转清晰）；支持优先级、SLA、当前处理人、超时高亮。
- **业主成员：** 档案（基本信息、家庭成员、车辆、宠物、缴费记录）+ **批量导入/导出**入口明确（主按钮级）。
- **账单：** 物业费、水电费、停车费等；列表与详情需支撑 **在线支付状态** 与 **对账** 视图。
- **图标：** Heroicons / Lucide，**禁用 emoji 当图标**。

## 工单流程（后台视角）

与 Master「产品上下文」一致，后台需支持筛选、批量操作与审计：

| 主状态 | 说明 |
|--------|------|
| 待受理 | 已提交，未分配 |
| 已派单 / 待接单 | 已指定执行人；物业 H5 可「接单」进入处理中 |
| 处理中 | 已接单，执行中 |
| 待确认 | 处理完成，待业主或管理员确认（可配置） |
| 已完成 | 已确认闭环 |
| 已关闭 | 无需继续跟进（合并单、无效单等） |
| 已取消 | 用户或管理员撤销 |
| 挂起/待协调（可选） | 等配件、等业主、等第三方 |

**类型差异简述：** 报修重派单与闭环；投诉重分级与答复；巡检重计划与异常转单；派单重指派与执行反馈。

## Recommendations

- 表格提供 **列定制、导出、关键列固定**；长时间操作使用 **抽屉或全屏页** 减少迷失。
- **键盘与焦点：** 后台以键盘操作为主的用户多，焦点环与快捷键提示可适当加强。
- 动画保持 **150～300ms**，尊重 `prefers-reduced-motion`。
